<task type="checkpoint:human-action" gate="blocking">
  <name>Checkpoint: Enable pgvector extension in Neon console</name>
  <action>
Enable the pgvector extension in the Neon database using the Neon console SQL editor. This is a one-time manual step — it cannot be run via the Vercel CLI.

Steps:
1. Open the Neon console at https://console.neon.tech (or navigate via the Vercel Storage tab → open in Neon)
2. Select the database connected to crowe-ai-onboarding
3. Open the SQL Editor
4. Run this exact SQL:
   ```sql
   CREATE EXTENSION IF NOT EXISTS vector;
   ```
5. Then run the verification query:
   ```sql
   SELECT * FROM pg_extension WHERE extname = 'vector';
   ```
   This must return exactly 1 row. If it returns 0 rows, the extension did not enable — check that you are connected to the correct database.

Do NOT create any tables or indexes here — that is Phase 2's job. Do NOT create an IVFFlat index here — that belongs in Phase 3 after data is loaded.
  </action>
  <resume-signal>Type "done" when the SELECT query returns 1 row, or paste the error message if it failed</resume-signal>
